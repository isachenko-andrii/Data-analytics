## 📐 Статистика та EDA

| Термін | Опис / Розшифровка | Ключовий контекст |
| :--- | :--- | :--- |
| **A/B Тестування (A/B Testing)** | Метод статистичного дослідження, де порівнюються два варіанти (контрольний $A$ та тестовий $B$) для перевірки впливу змін на цільові метрики. | Перевірка гіпотез, p-value |
| **Аномалія / Викид (Outlier)** | Спостереження в датасеті, яке лежить на аномальній відстані від інших значень і суттєво відхиляється від загального розподілу. | Очищення даних, IQR, Z-score |
| **Вінзоризація (Winsorization)** | Спосіб обробки викидів, за якого екстремальні значення замінюються найближчими «допустимими» (наприклад, на рівні 1-го чи 99-го перцентиля). | Очищення даних, Robust Stats |
| **Генеральна сукупність (Population)** | Повна множина всіх можливих об'єктів або спостережень, які є предметом дослідження. | Статистичний висновок |
| **Дисперсія (Variance)** | Міра розсіювання значень випадкової величини навколо її середнього значення (математичного сподівання). | Статистичний аналіз |
| **Довірчий інтервал (Confidence Interval)** | Діапазон значень, який з певною заданою ймовірністю (наприклад, 95%) містить істинний параметр генеральної сукупності. | Інференційна статистика |
| **Інтерквартильний розмах (IQR)** | Різниця між 75-м (Q3) та 25-м (Q1) квартилями. Використовується для побудови Box plot та пошуку викидів ($Q3 + 1.5 \times IQR$). | Описова статистика, EDA |
| **Кореляція (Correlation)** | Статистичний зв'язок між двома або більше змінними (напр., Пірсона чи Спірмена). *Важливо: Кореляція не означає причинно-наслідкового зв'язку.* | EDA, Мультиколінеарність |
| **Кумулятивний ефект (Cumulative Effect)** | Накопичувальний результат послідовних подій або змін показників за певний проміжок часу. | Аналіз часових рядів |
| **Медіана (Median)** | Значення, яке ділить упорядкований набір даних навпіл. На відміну від середнього, медіана стійка (robust) до викидів. | Описова статистика |
| **Мода (Mode)** | Значення або категорія, яка найчастіше зустрічається в аналізованому наборі даних. | Описова статистика |
| **Мультиколінеарність** | Явище високої кореляції між незалежними змінними (предикторами) у регресійному аналізі, що дестабілізує оцінки коефіцієнтів. | Лінійна регресія, Ознаки |
| **Нормальний розподіл (Gaussian)** | Симетричний дзвоноподібний розподіл ймовірностей, де більшість значень групується навколо середнього (правило 3-х сігм). | Параметричні тести |
| **p-value (Ймовірність значущості)** | Ймовірність отримати поточні або ще більш екстремальні результати тесту за умови, що нульова гіпотеза ($H_0$) є абсолютно правильною. | Перевірка гіпотез, $\alpha = 0.05$ |
| **Розвідницький аналіз (EDA)** | Етап дослідження датасету без жорстких початкових гіпотез з метою візуалізації, пошуку закономірностей, аномалій та структури даних. | Pandas, Seaborn, Matplotlib |
| **Середньоквадратичне відхилення ($\sigma$)** | Показник розсіювання даних, що дорівнює квадратному кореню з дисперсії. Показує середню відстань точок від середнього значення. | Описова статистика |
| **Семплювання (Sampling)** | Процес формування репрезентативної вибірки з генеральної сукупності для зниження витрат на обчислення або проведення тесту. | Статистичний аналіз |
| **Центральна гранична теорема (CLT)** | Теорема, яка стверджує, що розподіл середніх значень вибірок наближається до нормального із зростанням розміру вибірки, незалежно від форми початкового розподілу. | Основа А/В тестування |

## 🡪 Інженерія даних та ETL/ELT

| Термін | Опис / Розшифровка | Ключовий контекст |
| :--- | :--- | :--- |
| **Агрегація даних (Data Aggregation)** | Процес збору, групування та математичного узагальнення сирих даних (наприклад, `SUM`, `AVG`, `COUNT` в SQL). | Обробка даних, OLAP |
| **Валідація даних (Data Validation)** | Автоматична або ручна перевірка даних на відповідність бізнес-правилам, типам даних, обмеженням цілісності та унікальності. | Data Quality, Great Expectations |
| **Датасет (Dataset)** | Будь-яка структурована сукупність даних (таблиця, матриця, колекція JSON-документів), готова для обробки. | Обробка даних |
| **Лог-файли (Log Files)** | Хронологічні файли записів системних подій, транзакцій або дій користувачів, що слугують першоджерелом для аудиту та поведінкового аналізу. | Збір даних, Клікстрім |
| **Очищення даних (Data Cleaning)** | Процес виявлення та виправлення помилок, видалення дублікатів, обробки пропущених значень ($NaN$) та невалідних записів. | Pandas, PySpark |
| **Парсинг / Скрейпінг (Scraping)** | Автоматизоване вилучення неструктурованих даних із веб-сторінок та перетворення їх на структуровані формати (CSV, JSON, SQL). | BeautifulSoup, Scrapy, API |
| **Сховище даних (DWH)** | Централізована реляційна або колоночна система зберігання інтегрованих історичних даних компанії, оптимізована для аналітичних запитів. | Snowflake, BigQuery, ClickHouse |
| **Трансформація даних** | Етап перетворення типів даних, нормалізації, денормалізації, кодування категоріальних ознак та створення розрахункових полів. | Етап "T" в ETL |
| **API (Application Interface)** | Набір протоколів, за допомогою яких аналітичні системи та скрипти можуть автоматизовано забирати дані з першоджерел або сервісів. | Збір даних, Requests |
| **Data Lake (Озеро даних)** | Сховище для зберігання величезних обсягів сирих даних у їхньому початковому форматі (структуровані, напівструктуровані, неструктуровані). | AWS S3, Hadoop HDFS |
| **Data Lineage** | Процес візуалізації та відстеження життєвого циклу даних: від джерела виникнення, через усі трансформації, до фінального звіту. | Data Governance, dbt |
| **ELT (Extract, Load, Transform)** | Сучасний підхід до обробки даних, коли сирі дані спочатку завантажуються в потужне хмарне сховище, а вже потім трансформуються силами DWH. | Хмарні DWH, dbt, Airflow |
| **ETL (Extract, Transform, Load)** | Класичний процес управління даними: вилучення з джерел, трансформація на проміжному сервері (Staging) та завантаження в DWH. | Інженерія даних, Talend |
| **SQL (Structured Query Language)** | Мова декларативного програмування, що використовується для створення, модифікації та отримання даних із реляційних баз даних. | PostgreSQL, MySQL, T-SQL |

## 📈 Продуктова та Маркетингова аналітика

| Термін | Опис / Розшифровка | Ключовий контекст |
| :--- | :--- | :--- |
| **Атрибуція (Attribution)** | Модель розподілу цінності (конверсії) між різними каналами маркетингу, з якими взаємодіяв користувач (First Click, Last Click, Data-Driven). | Маркетингова аналітика |
| **Вирва конверсії (Funnel)** | Концептуальна модель, яка відображає етапи подорожі клієнта від першої взаємодії (точки входу) до цільової дії (покупки). | Продуктовий аналіз, Google Analytics |
| **Когортний аналіз (Cohort Analysis)** | Метод аналізу, при якому користувачі групуються за часом здійснення певної події (тиждень реєстрації тощо) для відстеження їхньої поведінки в динаміці. | Утримання, Retention Rate |
| **Частота відтоку (Churn Rate)** | Відсоток клієнтів або підписників, які припинили взаємодію з компанією чи продуктом за визначений проміжок часу. | Продуктові метрики, SaaS |
| **AARRR (Pirate Metrics)** | Продуктовий фреймворк, що описує життєвий цикл клієнта: Acquisition (Залучення), Activation (Активація), Retention (Утримання), Referral (Вірусність), Revenue (Дохід). | Продуктовий менеджмент |
| **ARPU / ARPPU** | Average Revenue Per (Paying) User. Середній дохід на одного (платячого) користувача за фіксований період. | Монетизація |
| **CAC (Customer Acquisition Cost)** | Повна сума витрат на маркетинг і продажі, необхідна для залучення одного нового клієнта. | Економіка одиниці (Unit Economics) |
| **LTV / CLV (Lifetime Value)** | Прогнозний або фактичний сукупний чистий прибуток, який компанія отримує від одного клієнта за весь період його життєвого циклу. | Бізнес-стратегія |
| **NPS (Net Promoter Score)** | Індекс лояльності клієнтів. Вимірюється на основі питання: "Яка ймовірність того, що ви порекомендуєте нас друзям?", ділить клієнтів на Промоутерів, Скептиків і Критиків. | Клієнтський досвід (CX) |
| **Retention Rate (Коефіцієнт утримання)** | Відсоток користувачів, які повертаються в продукт через $N$ днів після моменту їхньої першої активації/реєстрації. | Головна метрика продукту |

## 🤖 Моделювання та Machine Learning

| Термін | Опис / Розшифровка | Ключовий контекст |
| :--- | :--- | :--- |
| **Кластеризація (Clustering)** | Алгоритм навчання без вчителя (Unsupervised Learning) для автоматичного групування об'єктів на основі їхньої математичної схожості. | K-Means, Сегментація клієнтів |
| **Лінійна регресія (Linear Regression)** | Модель, яка прогнозує значення безперервної залежної змінної шляхом пошуку оптимального лінійного рівняння зв'язку з предикторами. | Предиктивна аналітика, $Y = wX + b$ |
| **Логістична регресія** | Модель класифікації, яка використовує логістичну функцію (сигмоїду) для прогнозування ймовірності приналежності об'єкта до певного класу. | Бінарна класифікація |
| **Нормалізація / Масштабування** | Приведення числових ознак до єдиного масштабу (наприклад, [0, 1] або стандартизація із середнім 0 і відхиленням 1) для стабільної роботи алгоритмів. | MinMaxScaler, StandardScaler |
| **Предиктивна аналітика** | Галузь аналітики, що використовує історичні дані, статистичне моделювання та машинні алгоритми для прогнозування майбутніх трендів і подій. | Прогнозування |
| **Часовий ряд (Time Series)** | Послідовність точок даних, зібраних через рівні проміжки часу. Має специфічні компоненти: тренд, сезонність, циклічність та шум. | ARIMA, Prophet, Прогнозування продажів |
| **Confusion Matrix (Матриця помилок)** | Таблиця, що використовується для оцінки якості роботи моделі класифікації. Відображає True Positive, False Positive, True Negative, False Negative значення. | Метрики якості (Precision, Recall, F1) |
| **Overfitting (Перенавчання)** | Ситуація, коли модель занадто ідеально підлаштувалася під тренувальні дані (включаючи випадковий шум), через що втратила здатність до генералізації на нових даних. | Валідація моделі, Регуляризація |

## 🖥 BI, Дашборди та Управління даними

| Термін | Опис / Розшифровка | Ключовий контекст |
| :--- | :--- | :--- |
| **Бізнес-аналітика (BI)** | Набір технологій, інструментів та практик для збору, інтеграції, аналізу та представлення бізнес-інформації у вигляді звітів. | Power BI, Tableau, Looker |
| **Візуалізація даних** | Мистецтво та технологія представлення числових даних у графічному вигляді (діаграми, карти, графи) для миттєвого зчитування інсайтів нетехнічними стейкхолдерами. | Data Storytelling |
| **Дашборд (Dashboard)** | Інтерактивна, візуальна аналітична панель, що консолідує метрики компанії, надаючи користувачам можливість фільтрації та деталізації (drill-down). | BI розробка |
| **Метрика (Metric)** | Стандартизований кількісний показник, що відображає стан конкретного процесу чи бізнес-активності. | KPI, OKR |
| **Data Governance** | Комплексна система управління даними в компанії: визначення прав доступу, стандартів якості, безпеки, відповідності регуляціям (GDPR тощо). | Менеджмент даних |
| **Data Mesh** | Сучасний децентралізований архітектурний підхід до управління даними, де дані розглядаються як продукт (Data Product) і контролюються конкретними бізнес-доменами. | Архітектура великих компаній |
| **Data Storytelling** | Практика побудови зв'язної розповіді навколо аналітичних даних, що поєднує в собі самі дані, візуалізацію та контекстний наратив для спонукання до рішень. | Презентація звітів |
